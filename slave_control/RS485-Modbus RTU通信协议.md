### 一、 物理层与链路层基础规格
+ **物理接口**：RS485 半双工（单总线）
+ **通信协议**：标准 Modbus RTU
+ **波特率**：**19200bsp** **115200 bps**（因从机多达26个，9600bps会导致轮询周期过长）
+ **数据格式**：8 数据位，无校验 (None)，1 停止位 (8-N-1)
+ **字节序**：大端模式 (Big-Endian，高字节在前，低字节在后)
+ **数据缩放**：涉及小数的浮点数（如温度 24.6℃、EC值 2.34）在传输时均放大 10 倍或 100 倍转为 16位整数 (INT16/UINT16) 传输。

| Start | Address | Function | Data | CRC | End |
| --- | --- | --- | --- | --- | --- |
| 3.5 Char time | 8 Bit | 8 Bit | N * 8Bit | 16 Bit | 3.5 Char time |


RTU 的本质依赖**时间间隔**：

```plain
帧间隔 ≥ 3.5 字符时间
```

##### 字符时间计算
$ T_{char} = \frac{1}{baudrate} \times (start + data + parity + stop) $

例如：

```plain
9600 baud → ≈ 1ms/字符
```

👉 帧间隔 ≈ 3.5ms

```latex
主站 → 请求读取一段寄存器
从站 → 返回对应数据
```

#### 1️⃣ 请求帧
```latex
[从站地址][0x03][起始地址][数量][CRC]
```

##### 示例
```latex
01 03 00 00 00 02 C4 0B
```

解析：

| 字段 | 值 | 含义 |
| --- | --- | --- |
| 地址 | 01 | 从站1 |
| 功能码 | 03 | 读寄存器 |
| 起始地址 | 0000 | 从0开始 |
| 数量 | 0002 | 读2个寄存器 |
| CRC | C4 0B | 校验 |


#### 2️⃣ 响应帧
```latex
[地址][0x03][字节数][数据][CRC]
```

##### 示例响应
```latex
01 03 04 00 0A 01 02 CRC
```

解析：

| 字段 | 含义 |
| --- | --- |
| 04 | 4字节（2个寄存器） |
| 00 0A | 寄存器0 |
| 01 02 | 寄存器1 |

### 四、 举例说明
在使用 **Modbus Poll** 工具进行调试时，请先确保串口连接参数设置正确：**波特率 19200/115200，8个数据位，无校验 (None)，1个停止位**。

以下示例中，所有的报文数据均为 **十六进制 (Hex)** 格式。为了方便阅读，最后的 CRC 校验码用 `[CRC低] [CRC高]` 代替（Modbus Poll 等工具会自动计算并追加 CRC）。

#### 示例 1：读取温控板的环境数据 (FC=0x04)
**目标**：主机读取温控板（从机 `0x01`）上的一层、二层、箱外温度，以及一层、二层、箱外湿度（共读取 6 个寄存器，地址 `0x0000` 到 `0x0005`）。

+ **Modbus Poll 设置**：
    - Slave ID: `1`
    - Function: `04 Read Input Registers`
    - Address: `0` (对应 Hex `0x0000`)
    - Quantity: `6`

**► 主机发送 (Tx)**

Plaintext

```plain
01 04 00 00 00 06 [CRC低] [CRC高]
```

_(解析：_`_01_`_从机，_`_04_`_读输入寄存器，_`_00 00_`_起始地址，_`_00 06_`_读取6个寄存器)_

**◄ 从机返回 (Rx)**

Plaintext

```plain
01 04 0C 00 F6 00 FB 00 F6 02 AA 02 A0 02 C1 [CRC低] [CRC高]
```

**数据解析**（返回了 0C 即 12 个字节的数据）：

+ `00 F6` (十进制 246)：一层温度 24.6 ℃ (缩小10倍)
+ `00 FB` (十进制 251)：二层温度 25.1 ℃
+ `00 F6` (十进制 246)：箱外温度 24.6 ℃
+ `02 AA` (十进制 682)：一层湿度 68.2 % (缩小10倍)
+ `02 A0` (十进制 672)：二层湿度 67.2 %
+ `02 C1` (十进制 705)：箱外湿度 70.5 %

#### 示例 2：控制一层制冷片1的 PWM 值 (FC=0x06)
**目标**：主机向温控板（从机 `0x01`）的制冷片1寄存器（地址 `0x0000`）写入 PWM 占空比值 1000。

+ **Modbus Poll 设置**：
    - 双击对应的寄存器地址，选择 `06 Write Single Register`
    - Value: `1000` (对应 Hex `0x03E8`)

**► 主机发送 (Tx)**

Plaintext

```plain
01 06 00 00 03 E8 [CRC低] [CRC高]
```

_(解析：_`_01_`_从机，_`_06_`_写单个寄存器，_`_00 00_`_寄存器地址，_`_03 E8_`_写入值1000)_

**◄ 从机返回 (Rx)**

Plaintext

```plain
01 06 00 00 03 E8 [CRC低] [CRC高]
```

_(解析：功能码 0x06 的标准应答是原样返回主机的发送帧，表示写入成功)_

#### 示例 4：批量下发植物灯 LED 1 的 5个通道光照策略 (FC=0x10)
**目标**：主机向光照板（从机 `0x03`）一次性写入 LED 1 的 R, W, B, NIR, UVB 五个通道的 PWM 值（写入地址 `0x0000` 开始的 5 个寄存器）。

假设要求：R=800 (`0x0320`), W=600 (`0x0258`), B=400 (`0x0190`), NIR=200 (`0x00C8`), UVB=0 (`0x0000`)。

+ **Modbus Poll 设置**：
    - Slave ID: `3`
    - Function: `16 Write Multiple Registers` (注：十六进制的0x10在软件里通常显示为十进制的16)
    - Address: `0`
    - Quantity: `5`

**► 主机发送 (Tx)**

Plaintext

```plain
03 10 00 00 00 05 0A 03 20 02 58 01 90 00 C8 00 00 [CRC低] [CRC高]
```

_(解析：_`_02_`_从机，_`_10_`_写多个寄存器，_`_00 00_`_起始地址，_`_00 05_`_写5个寄存器，_`_0A_`_共计10个数据字节，后面跟着具体的5个PWM值)_

**◄ 从机返回 (Rx)**

Plaintext

```plain
03 10 00 00 00 05 [CRC低] [CRC高]
```

_(解析：从机回应“我收到了发给起始地址 _`_0000_`_ 的 _`_0005_`_ 个寄存器的修改请求”)_

#### 示例 5：读取混液箱 EC 浓度 (FC=0x04)
**目标**：主机读取 EC 传感器（从机 `0x18`）的 EC 浓度值（读取 1 个寄存器，地址 `0x0000`）。

+ **Modbus Poll 设置**：
    - Slave ID: `24` (对应 Hex `0x0D`)
    - Function: `04 Read Input Registers`
    - Address: `0`
    - Quantity: `1`

**► 主机发送 (Tx)**

Plaintext

```plain
18 04 00 00 00 01 [CRC低] [CRC高]
```

**◄ 从机返回 (Rx)**

Plaintext

```plain
18 04 02 00 EA [CRC低] [CRC高]
```

**数据解析**（返回 02 即 2 个字节的数据）：

+ `00 EA` (十进制 234)：根据协议缩放 x100 规则，实际 EC 值为 2.34 ms/cm。
